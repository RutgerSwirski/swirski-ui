# AGENTS.md

Guidance for AI agents and automated coding assistants working on Swirski UI.

## Project Shape

- `packages/ui` is the React component package published as `@swirski/ui`.
- `apps/docs` is the docs site and public registry host.
- `registry/swirski.registry.json` is the source manifest for local and hosted registry workflows.
- `apps/docs/public/llms.txt` and `apps/docs/public/llms-full.txt` are generated by `pnpm llms:build`.

## Component Conventions

- Prefer small component folders with an `index.ts` barrel and explicit named exports from `packages/ui/src/index.ts`.
- Design public APIs around `variant`, `size`, `tone`, `className`, forwarded refs, and native HTML props.
- Use `asChild` with `Slot` only when polymorphic composition is genuinely useful.
- Keep controlled and uncontrolled APIs consistent for interactive components.
- Keep `className` escape hatches available, and add targeted class props only when a component has distinct internal surfaces.

## Styling Conventions

- Use `cn` for class composition.
- Use `swirskiAttrs(component, { size, tone, variant })` on public component surfaces.
- Prefer theme tokens such as `--sw-color-ink`, `--sw-color-surface`, `--sw-color-shadow`, `--sw-color-focus`, `--sw-border-width`, and `--sw-shadow-*`.
- Prefer shared interaction helpers such as `focusVisibleStyles` and `disabledInteractiveStyles`.
- Avoid one-off hardcoded styles when an existing token or local component pattern is available.
- Preserve the Swirski visual language: thick borders, hard offset shadows, graphic color, uppercase labels, and compact practical layouts.

## Accessibility Expectations

- Prefer native HTML behavior where it keeps the component simpler and more reliable.
- Forward refs on public components.
- Preserve semantic roles, labels, `aria-*` relationships, and keyboard behavior.
- Complex overlays should handle Escape, outside pointer interactions, focus restore, and focus containment where appropriate.
- Floating content should use the shared Floating UI helper in `packages/ui/src/system/floating.ts`.
- Modal content should use the shared overlay helper in `packages/ui/src/system/overlay.ts`.
- Add or update tests for keyboard behavior, focus behavior, disabled states, controlled/uncontrolled state, and accessible names when touching interaction primitives.

## Docs And Registry

- When adding a component, update the package export, docs metadata, playground metadata when relevant, and `registry/swirski.registry.json`.
- When adding a hook, update the hook barrel, docs metadata, and registry manifest.
- Run `pnpm docs:metadata`, `pnpm llms:build`, and `pnpm registry:shadcn` after metadata or registry changes.
- Generated files under `apps/docs/public` should be regenerated rather than edited by hand.

## Verification

- For UI package changes, run `pnpm test:ui` and `pnpm build:ui` when feasible.
- For docs or registry changes, run `pnpm docs:metadata`, `pnpm llms:build`, and `pnpm registry:shadcn`.
- Keep changes scoped. Do not refactor unrelated components while fixing a single component.
